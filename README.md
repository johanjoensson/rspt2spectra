# rspt2spectra

[![CI](https://github.com/johanjoensson/rspt2spectra/actions/workflows/ci.yml/badge.svg)](https://github.com/johanjoensson/rspt2spectra/actions/workflows/ci.yml)
[![Coverage Status](https://coveralls.io/repos/github/johanjoensson/rspt2spectra/badge.svg?branch=master)](https://coveralls.io/github/johanjoensson/rspt2spectra?branch=master)
[![Documentation](https://img.shields.io/badge/docs-github%20pages-blue)](https://johanjoensson.github.io/rspt2spectra/)

This package reads [RSPt](http://fplmto-rspt.org/) output and turns real-frequency
hybridization functions into non-interacting Anderson impurity Hamiltonians (h0),
including bath orbitals. It owns the full pipeline: RSPt files → Δ(ω) analysis → block
partition (`block_structure`) → bath fit (`hyb_fit`, `natural_orbitals`) → bath geometry
— star, chains, or linked double chain (`edchain`) → h0 assembly (`h0`). The resulting
h0 can be consumed by any many-body impurity solver, e.g. the
[impurityModel](https://github.com/johanjoensson/impurityModel) software (see `h2imp`
for writing files in its operator format); this package does not depend on any solver.

## Installation

Requires Python 3.11+. Runtime dependencies are only `numpy` and `scipy`.

```bash
git clone https://github.com/johanjoensson/rspt2spectra.git
cd rspt2spectra
pip install .
```

Optional extras:

```bash
pip install .[mpi]    # MPI-parallel fits (requires a system MPI, e.g. OpenMPI)
pip install .[plot]   # matplotlib, for build_h0 --plot
pip install -e .[dev]  # development: pytest, coverage, ruff
pip install -e .[docs] # building the documentation
```

## Quick start

In an RSPt simulation directory where `rspt` has been run with

```
spectrum
Dos Hyb
```

in `green.inp` (so that `real-/imag-hyb-*.dat` exist and the local Hamiltonian is
printed in the `out` file), run:

```bash
build_h0 <cluster> <bath_states_per_orbital>
```

e.g. `build_h0 0102010100 10`. This fits the hybridization function and writes the
non-interacting Hamiltonian in operator format to `<cluster>_h0_op.dict`. With
`mpi4py` installed, run under MPI to parallelize over initial guesses:

```bash
mpiexec -n 8 build_h0 <cluster> <bath_states_per_orbital>
```

See `build_h0 --help` and the
[documentation](https://johanjoensson.github.io/rspt2spectra/) for the full workflow,
including how to choose impurity orbitals in RSPt and how to extract Slater-Condon
integrals and SOC parameters for the downstream impurity-model calculation.

Note that values generated by RSPt are in Rydberg; this package works in whatever
unit its input files use.

## Development

```bash
pip install -e .[dev]
pytest                       # run the test suite
pytest --cov=rspt2spectra    # with coverage
ruff check src tests         # lint
ruff format src tests        # format
```

Documentation is built with Sphinx:

```bash
pip install -e .[docs]
sphinx-build -W -b html docs docs/_build/html
```

Continuous integration runs the test suite on Python 3.11–3.14 (serially and under
MPI), checks lint/formatting, uploads coverage to Coveralls, and builds and deploys
the documentation to GitHub Pages on pushes to `master`.
