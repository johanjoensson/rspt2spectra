Usage
=====

The typical workflow has two sides: prepare the RSPt simulation so that it
prints the quantities this package needs, then run the ``build_h0`` console
script in the simulation directory to fit the hybridization function and
assemble the non-interacting Hamiltonian.

.. note::

   Values generated by RSPt are in Rydberg; ``rspt2spectra`` operates on
   whatever unit the input files use and does not convert.

Preparing the RSPt simulation
-----------------------------

Choose impurity orbitals
^^^^^^^^^^^^^^^^^^^^^^^^

For the hybridization function and the on-site energies of the impurity, the
choice of impurity orbitals matters.

* For an atom in a cubic environment (O\ :sub:`h` symmetry) the natural
  choice is cubic harmonics: use basis index ``3`` in the ``green.inp``
  cluster definition.
* For low-symmetry systems, RSPt's default spherical harmonics
  (basis index ``0``) are a convenient choice. ``build_h0`` automatically
  block-partitions the hybridization function and local Hamiltonian, so a
  diagonal basis is not required.

Generate the hybridization function
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Add the block::

   spectrum
   Dos Hyb

to the ``green.inp`` file and run ``rspt`` once. This produces the
``real-hyb-*.dat`` / ``imag-hyb-*.dat`` files and prints the local
Hamiltonian to the ``out`` file (grep for ``Local hamiltonian``).

Building h0
-----------

Run ``build_h0`` in the RSPt simulation directory::

   build_h0 <cluster> <bath_states_per_orbital>

where ``<cluster>`` is the RSPt cluster label (e.g. ``0102010100``) and
``<bath_states_per_orbital>`` is the average number of bath states to fit
per orbital block. The script

1. reads the hybridization function (``real-/imag-hyb-<cluster>.dat``) and
   the local Hamiltonian and crystal-field transformation from ``out``,
2. block-partitions the hybridization function,
3. fits star bath energies and hoppings per inequivalent block
   (:func:`rspt2spectra.hyb_fit.fit_hyb`, or
   :func:`rspt2spectra.natural_orbitals.fit_hyb_natural_orbitals` with
   ``--natural-orbitals``),
4. transforms the bath to the requested geometry (``--bath-geometry``:
   star, chains, or linked double chain, see :mod:`rspt2spectra.edchain`),
5. writes the resulting h0 operator to ``<cluster>_h0_op.dict``.

Useful options (see ``build_h0 --help`` for the full list):

``--bath-geometry <geometry>``
   Bath geometry to build (default ``Star``).
``--eim <value>``
   Distance above the real axis at which the hybridization function is
   evaluated (default 0.01).
``--fit-unocc``
   Also fit the unoccupied (positive-energy) part of the hybridization
   function; by default only the occupied part is fitted.
``--weight-function {unit,exponential,gaussian,lorentzian}``, ``--weight-factor``, ``--fit-center``
   Weighting of the fit as a function of energy, see
   :mod:`rspt2spectra.weight_functions`.
``--regularization {l1,l2}``
   Regularization applied to the fitted hopping parameters.
``--natural-orbitals``
   Use the natural-orbitals discretization instead of non-linear fitting.
``-p, --plot``
   Plot the fitted hybridization function against the input (requires
   matplotlib).

MPI parallelism
^^^^^^^^^^^^^^^

When ``mpi4py`` is installed, the fit can be run in parallel; each rank
starts from different initial guesses and the best fit is kept::

   mpiexec -n 8 build_h0 <cluster> <bath_states_per_orbital>

Using the result
----------------

The generated ``<cluster>_h0_op.dict`` file contains the one-particle
Hamiltonian in second-quantization operator format,
``{((i, "c"), (j, "a")): h0[i, j]}``, directly consumable by e.g.
`impurityModel <https://github.com/johanjoensson/impurityModel>`_.
Interacting parameters (Slater-Condon integrals, SOC) are not part of h0;
extract them from RSPt as described below.

Extracting Slater-Condon integrals from RSPt
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

If some of the correlated orbitals of interest are treated as core states in
RSPt, first create a subdirectory containing a copy of the RSPt simulation
directory and move the core states into the valence energy set by modifying
the ``data`` file.

To make RSPt print Slater-Condon integrals, add a cluster in ``green.inp``
containing the correlated orbitals of interest, e.g.::

   # 3d and 2p of type 1
   cluster
   2 1 2 eV
   1 2 1 1 0 -1.0
   1 1 3 1 0 -1.0
   0 0 0.30

and also add::

   verbose
   Umatrix

   debug
   Noscreening

Run RSPt once; the Slater-Condon integral values are printed to the ``out``
file (grep for ``Slater parameters``).

Extracting SOC parameters from RSPt
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

The SOC parameters are printed in the RSPt-generated ``out`` file.

* For orbitals treated as core states, the orbital energy difference is
  found by grepping for ``core states``. For example, for 2p orbitals the
  energy difference equals 3/2 times the SOC parameter.
* For orbitals treated as valence states, the SOC parameter can be extracted
  if the ``data`` file has ``f-rel`` set to ``t``. A ``green.inp`` with a
  cluster of the orbitals of interest is needed; the local Hamiltonian is
  printed in the ``out`` file (grep for ``Local hamiltonian``) and the SOC
  parameter equals its (2,6) element when spherical harmonics are used.
